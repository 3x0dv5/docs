.. _bugs:

*****************************************************************************
 Soumission de bug
*****************************************************************************

Les bugs/tickets doivent être soumis via through github (https://github.com/mapserver/mapserver/issues). 
Vous devez d'abord vous créer un compte personnel gratuit sur github (https://github.com/plans).

Garder en tête les informations suivantes lors de la soumission d'un ticket : 

1. S'il vous plaît définissez le *Label* soigneusement. Il déterminera à qui le ticket sera 
  assigné par défaut.

 .. note::
     Tous les bugs sur sécurité/vulnérabilité doivent utiliser le label 
     **Security/Vulnerability (Private)**. Il est important d'utiliser 
     ce label pour ces bugs afin de s'assurer que le mainteneur des bugs 
     de sécurité soit au courant du problème et que la bonne procédure 
     soit mise en place.
	 
2. Définissez une brève description compréhensible. 

3. En général vous ne devez pas assigner un utilisateur spécifique (les 
   développeurs appropriés seront notifié de votre problème via le 'label').

4. Dans votre description indiqué s'il vous plait si vous compilez des sources 
   ou à partir d'un binaire préparé, et spécifiez la version de MapServer.

La chose la plus importante lors d'un rapport de but est de fournir un 
exemple minimal qui est nécessaire pour reproduire le bug. Cela signifie un 
mapfile minimale et les données auquel il dépend. Supprimez tout ce qui n'est 
pas nécessaire pour reproduire le bug.

Les développeurs souvent n'aime pas perdre du temps les 30 premières minutes 
lorsqu'ils travaillent sur un bug pour fixer les chemins, supprimer les couches 
inutiles, supprimer les références vers des symbols ou police externes qui ne sont 
pas inclus ou pas nécessaire et d'une manière générale analyser votre test pour 
le faire fonctionner pour.

Si le bug est facilement visible avec ":ref:`shp2img`", sans avoir un service web 
et à le tester via http, alors indiquez le. Si un script :ref:`mapscript` 
indépendant peut montrer un problème sans être un service web, soumettez le de cette manière.

Le temps qu'un bug soit corrigé rapidement est directement lié au temps que va 
mettre le développeur pour reproduire le problème. Si cela est difficile, la 
problématique qu'il soit laissé de côté ou ignoré est important.
